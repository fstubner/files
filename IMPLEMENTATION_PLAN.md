# AI-Powered File Explorer - Implementation Plan

## Project Overview

A high-performance, cross-platform file explorer that combines powerful traditional file management with optional AI-powered features. Built with modern best practices and flexibility in mind, users can choose to use it as a feature-rich traditional file explorer or enhance their experience with AI capabilities.

## AI Architecture Clarification

### Python AI Services vs Local AI Models

**Python AI Services** (Separate Process):
- Runs as a FastAPI microservice
- Handles complex AI operations that require Python libraries
- Can be disabled entirely for non-AI users
- Communicates with main app via HTTP/IPC

**What Python Services Handle**:

**Local AI Models in Rust** (Embedded):
- Runs directly in the Tauri app using Candle/ONNX Runtime
- For lightweight, fast operations
- No separate process needed

**When to Use Each**:
- **Python Services**: Heavy AI tasks (object detection, OCR, transcription)
- **Rust Local Models**: Fast operations (embeddings, similarity, basic classification)
- **External APIs**: Advanced language tasks (via Ollama, OpenRouter)

## Core Architecture Decisions

### Language Choice Analysis

**Option 1: Rust (Recommended for Core)**
- **Pros**: Blazing fast file operations, memory safety, excellent for system-level tasks
- **Cons**: Steeper learning curve
- **Performance**: 10/10

**Option 2: JavaScript/TypeScript + Rust Hybrid (Recommended Approach)**
```
Frontend: Vue.js (JavaScript/TypeScript)
Backend:
  - Core file operations: Rust (via Tauri)
  - Business logic: TypeScript (Node.js integration)
  - AI features: Python microservices (optional)
```
- **Pros**: Best of all worlds, easier development, modular
- **Cons**: Slightly more complex architecture
- **Performance**: 8-9/10

**Option 3: Full JavaScript/TypeScript**
- **Pros**: Unified language, faster development
- **Cons**: Slower file operations, higher memory usage
- **Performance**: 6-7/10 (still acceptable with optimization)

```typescript
// stores/contextMenu.ts
export const useContextMenuStore = defineStore('contextMenu', () => {
  const defaultItems: ContextMenuItem[] = [
    {
      id: 'copy',
      label: 'Copy',
      icon: 'mdi:content-copy',
      shortcut: 'Ctrl+C',
      enabled: true,
      action: 'file:copy'
    },
    // ... more defaults
  ]

  const customItems = useStorage('custom-context-items', [] as ContextMenuItem[])

  const menuItems = computed(() => {
    return [...defaultItems, ...customItems.value]
      .filter(item => !item.hidden)
      .sort((a, b) => a.order - b.order)
  })

  function addCustomItem(item: Partial<ContextMenuItem>) {
    const newItem: ContextMenuItem = {
      id: nanoid(),
      enabled: true,
      visible: true,
      ...item
    }
    customItems.value.push(newItem)
  }

  return {
    menuItems,
    addCustomItem,
    removeItem,
    updateItem,
    reorderItems
  }
})
```
